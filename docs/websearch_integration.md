# DeepShell WebSearch 实时搜索功能集成文档

## 📝 功能概述

DeepShell AI 现已集成实时网络搜索功能，能够自动识别用户查询的时效性需求，调用 webSearchMCP 服务获取最新信息，并将搜索结果智能注入到 AI 对话上下文中，提供准确、及时的回答。

## 🚀 核心特性

### 1. 自动搜索触发
- **智能查询识别**：自动识别需要实时信息的查询
- **关键词检测**：支持中英文关键词触发搜索
- **触发词汇**：
  - 中文：搜索、查找、最新、新闻、当前、现在、今天、最近、什么是、如何、怎么、为什么
  - 英文：search、find、latest、news、current、now、today、recent、what is、how to、why、where、when

### 2. 多引擎搜索支持
- **Bing**：主要搜索引擎
- **DuckDuckGo**：隐私友好搜索
- **百度**：中文内容搜索
- **CSDN**：技术内容搜索
- **Brave**：去中心化搜索
- **Exa**：AI 优化搜索

### 3. 智能结果处理
- **UTF-8 编码优化**：完美支持中文搜索结果
- **结果格式化**：标准化搜索结果呈现
- **上下文注入**：搜索结果智能融入 AI 对话
- **来源标注**：明确标注信息来源和时效性

### 3. System Prompt 优化

**更新前**：

**更新后**：

```
你是一个友好、专业的AI助手，具备访问实时信息的能力。请遵循以下原则：

1. **优先使用提供的信息**：如果用户消息中包含"网络搜索结果"或"知识库信息"等参考资料，请优先基于这些信息回答问题。

2. **整合多个信息源**：当有多个搜索结果时，请综合分析并提供完整、准确的回答。

3. **标注信息来源**：在回答中适当提及信息来源，如"根据最新搜索结果"、"据报道"等。

4. **保持时效性**：对于新闻、股价、天气等时效性问题，重点关注最新的搜索结果。

5. **承认局限性**：只有在没有相关参考信息时才说"我不知道"，但要说明是因为缺乏相关信息。
```

## 🔍 故障排除

### 常见问题

1. **中文搜索结果乱码**
   - ✅ 已修复：实现了 `_fix_encoding()` 方法
   - 解决方案：Latin-1 到 UTF-8 的编码转换

2. **搜索未触发**
   - 检查查询是否包含触发关键词
   - 验证 `is_search_query()` 方法逻辑

3. **MCP 服务连接失败**
   - 确认服务运行在端口 3000
   - 检查防火墙和网络配置

4. **AI 不使用搜索结果**
   - ✅ 已修复：更新了 System Prompt
   - 确认上下文注入正常工作
